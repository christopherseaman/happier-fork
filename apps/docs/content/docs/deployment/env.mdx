This page lists the most common environment variables used by the server.

## Core

- `DATABASE_URL`: database connection string (Postgres/MySQL) for full flavor.
- `HANDY_MASTER_SECRET`: master key for auth tokens and server-side encryption.
- `HAPPIER_DB_PROVIDER`: `postgres` | `mysql` | `pglite` | `sqlite` (defaults depend on flavor).
- `HAPPIER_FILES_BACKEND`: `s3` (default) | `local`.
- `HAPPIER_SOCKET_ADAPTER`: `memory` (default) | `redis-streams`. Use `redis-streams` for multi-replica realtime fanout (requires `REDIS_URL`).
- `REDIS_URL`: Redis connection string, required when `HAPPIER_SOCKET_ADAPTER=redis-streams` (example: `redis://127.0.0.1:6379`).

## Voice (optional)

Server-minted ElevenLabs conversation tokens are exposed via `POST /v1/voice/token` and advertised via `GET /v1/features`.

Required for **Happier Voice**:

- `ELEVENLABS_API_KEY`
- `ELEVENLABS_AGENT_ID_PROD`

Required when subscription gating is enabled:

- `REVENUECAT_SECRET_KEY`

Optional controls:

- `VOICE_ENABLED` (`true`/`false`, default `true`)
- `VOICE_REQUIRE_SUBSCRIPTION` (`true`/`false`, defaults to `true` when `NODE_ENV=production`)
- `VOICE_FREE_SESSIONS_PER_MONTH` (default `0`)
- `VOICE_FREE_MINUTES_PER_MONTH` (default `0`, enforced when `VOICE_REQUIRE_SUBSCRIPTION=true`)
- `VOICE_MAX_CONCURRENT_SESSIONS` (default `1`)
- `VOICE_MAX_SESSION_SECONDS` (default `1200`, min `30`)
- `VOICE_MAX_MINUTES_PER_DAY` (default `0` = unlimited; global per-user guardrail)
- `VOICE_TOKEN_MAX_PER_MINUTE` (default `10`, `0` disables rate limiting)
- `VOICE_COMPLETE_MAX_PER_MINUTE` (default `60`, `0` disables rate limiting)
- `VOICE_LEASE_CLEANUP` (`true`/`false`, default `false`): periodically prune expired `VoiceSessionLease` rows.
- `VOICE_LEASE_RETENTION_DAYS` (default `30`, clamp 7–365): how long to retain expired leases.
- `VOICE_LEASE_CLEANUP_INTERVAL_MS` (default `21600000` = 6h, min `10000`): prune interval.

## Friends (optional)

Friends support is advertised via `GET /v1/features` and can be fully disabled on self-hosted servers.

Controls:

- `FRIENDS_ENABLED` (`true`/`false`, default `true`): master switch for Friends endpoints and UI.
- `FRIENDS_ALLOW_USERNAME` (`true`/`false`, default `false`): allows users to claim a local username (without GitHub).

Username policy (applies when `FRIENDS_ALLOW_USERNAME=true`):

- `FRIENDS_USERNAME_MIN_LEN` (default `3`)
- `FRIENDS_USERNAME_MAX_LEN` (default `32`)
- `FRIENDS_USERNAME_REGEX` (default `^[a-z0-9_-]+$`)

Notes:

- Usernames are normalized to **lowercase** by the server.
- If `FRIENDS_ALLOW_USERNAME=false`, Friends discovery expects users to connect GitHub (see below). Users may still be prompted to pick a different username if their GitHub login can’t be reserved (for example, it’s already taken).

## GitHub OAuth (optional)

GitHub OAuth is used to connect an account and (optionally) populate a username from the GitHub login.
The UI checks `GET /v1/features` to determine whether GitHub OAuth is configured.

Required:

- `GITHUB_CLIENT_ID`
- `GITHUB_CLIENT_SECRET`
- `GITHUB_REDIRECT_URL` (canonical) or `GITHUB_REDIRECT_URI` (legacy): should point to your server callback endpoint, e.g. `https://YOUR_SERVER/v1/oauth/github/callback`.

Optional:

- `HAPPIER_WEBAPP_URL` (default `https://app.happier.dev`): base URL/scheme for the client. The server redirects to `${HAPPIER_WEBAPP_URL}/oauth/github` after completing the OAuth callback (appends `?flow=connect|auth`, `?status=connected|username_required`, or `?error=...`). Legacy alias: `HAPPY_WEBAPP_URL`.
- `HAPPIER_WEBAPP_OAUTH_RETURN_URL_BASE` (optional): override the base URL for OAuth returns to the client, *without* the provider suffix.
  - Example: `https://app.example.com/custom-oauth` → server redirects to `https://app.example.com/custom-oauth/github?...`
  - If unset, the server derives it from `HAPPIER_WEBAPP_URL` (defaulting to `${HAPPIER_WEBAPP_URL}/oauth/${provider}`).
  - Legacy alias: `HAPPY_WEBAPP_OAUTH_RETURN_URL_BASE`.
- `HAPPIER_OAUTH_RETURN_ALLOWED_SCHEMES` (optional): allow additional OAuth return URL schemes (CSV/whitespace).
  - Default allows `https://` and loopback `http://localhost`.
  - Use this for mobile deep links, e.g. `myapp`.
  - Legacy alias: `HAPPY_OAUTH_RETURN_ALLOWED_SCHEMES`.
- `GITHUB_STORE_ACCESS_TOKEN` (`true`/`false`, default `false`): whether to persist the GitHub access token on the server. Friends does not require storing tokens. If you enable `AUTH_GITHUB_ALLOWED_ORGS` and set `AUTH_GITHUB_ORG_MEMBERSHIP_SOURCE=oauth_user_token`, the server defaults to storing tokens unless you explicitly set this to `false`.
- `GITHUB_HTTP_TIMEOUT_SECONDS` (default `10`, clamp 1–120): timeout for GitHub OAuth and membership HTTP calls.
- `OAUTH_PENDING_TTL_SECONDS` (default `600`, min `60`, max `3600`): how long a “pending OAuth finalize” (e.g. waiting for username selection) can be finalized.
  - Legacy alias: `GITHUB_OAUTH_PENDING_TTL_SECONDS` (if `OAUTH_PENDING_TTL_SECONDS` is unset).
- `OAUTH_STATE_TTL_SECONDS` (default `600`, min `60`, max `3600`): how long the OAuth `state` token is valid for (connect + provider signup).

## Auth policy (optional / enterprise)

Happier supports server-enforced **auth provider policies**. This is primarily useful for self-hosting and enterprise deployments that want to restrict:

- how accounts can be created (signup)
- which linked providers are required to keep using the server (login eligibility / offboarding)

### Signup / login methods

- `AUTH_ANONYMOUS_SIGNUP_ENABLED` (`true`/`false`, default `true`):
  - `true`: `POST /v1/auth` can create a new account for a never-before-seen device key.
  - `false`: new accounts must be created through an enabled external provider (e.g. GitHub).
- `AUTH_SIGNUP_PROVIDERS` (CSV, default empty): external providers allowed for **signup**, e.g. `github`.
- `AUTH_REQUIRED_LOGIN_PROVIDERS` (CSV, default empty): providers that must be linked (and eligible) for continued access, e.g. `github`.

### GitHub allowlists (optional)

When GitHub is required, you can restrict access further:

- `AUTH_GITHUB_ALLOWED_USERS` (CSV, default empty): allowlist of GitHub usernames (lowercased).
- `AUTH_GITHUB_ALLOWED_ORGS` (CSV, default empty): allowlist of GitHub orgs (lowercased).
- `AUTH_GITHUB_ORG_MATCH` (`any`/`all`, default `any`): how to evaluate multiple orgs.

### Offboarding (eligibility re-check)

If any allowlist is set, the server can periodically re-check eligibility and force logout when users are no longer eligible.

- `AUTH_OFFBOARDING_ENABLED` (`true`/`false`, default: `true` when any allowlist is set, else `false`)
- `AUTH_OFFBOARDING_INTERVAL_SECONDS` (default `86400`, min `60`, max `86400`)
- `AUTH_OFFBOARDING_STRICT` (`true`/`false`, default `false`): fail closed when eligibility cannot be re-checked due to upstream downtime (enterprise lock-down mode).

### GitHub org membership source (recommended: GitHub App)

To reliably enforce org membership without storing user OAuth tokens, use a **GitHub App**.

- `AUTH_GITHUB_ORG_MEMBERSHIP_SOURCE` (`github_app`/`oauth_user_token`):
  - defaults to `github_app` when `AUTH_GITHUB_ALLOWED_ORGS` is set
  - defaults to `oauth_user_token` otherwise

#### GitHub App configuration (when using `github_app`)

Required:

- `AUTH_GITHUB_APP_ID`
- `AUTH_GITHUB_APP_PRIVATE_KEY` (PEM)
- `AUTH_GITHUB_APP_INSTALLATION_ID_BY_ORG` (mapping, e.g. `acme=123,other=456`)

Notes:

- The GitHub App must have **Org Members: read** permission in each org you want to allowlist.
- The server uses installation tokens to call `GET /orgs/{org}/members/{username}` to validate membership.
